# Assignment description
The assignment involves applying the Normalized Systems (NS) theory that is taught in the course NIE-NSS. This
course covers the theoretical principles of NS theory, which will be put into practice in this assignment. The
learning outcome of the assignment is to bring the student’s understanding level of NS theory to higher levels:
evaluating and creating software that complies with NS theory.
The assignment is to select a comprehensive code base and review it based on NS based on NS principles:
modularity, evolvability, 4 theorems, etc. This code base can be code you wrote as part of your study program,
as part of a hobby project, or even a code base you are not involved in such as (part of) an open-source project.
The subject and programming language of the code base are free to choose. Next, you should refactor the code
base in such a way that it confirms to the NS theorems.
The scope of the assignment is 4 ECTS credits. According to the ECTS system, the assignment should therefore
equate to a workload of 100-120 working hours. The lecturers expect to assess submissions that reflect this
workload.

## Deliverables
1.A written report of a code review based on NS principles: modularity, evolvability, 4 theorems, etc. This
report also includes a description on how the code is refactored to comply with NS theory.
2.A refactored code base that complies to NS theory. The student refactors the code they have analyzed,
developing an implementation that complies with NS theory.
